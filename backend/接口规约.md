# 接口规约

## 客户端

### 轮询
#### url
/call/
#### 请求

方式：GET
<pre>
原始风格:
email=123456@qq.com&token=Vtv3232sd6...&request_time=2018-08-06_12:30:18
json风格:
{
    "email": "123456@qq.com",
    // token = 登录时服务器返回的token
    "token": "Vtv3232sd6...",
    // 请求时间
    "request_time": "2018-08-06_12:30:18"

}
</pre>
#### 回应

<pre>
{
    /*
    返回码
    0 for success
    1 for error
    */
    "code": "0",
    "email": "123456@qq.com",
    "msg": "success",
    /*
    请求状态
    "None": "无新消息"
    "Active": "有新消息"
    "error": "出现错误"
    */
    // status 为"None"或"error"时无对应列表属性
    "message_status": "Active",
    "message_active": [
        {
            "sender_name": "alien",
            "sender_email": "123@q.com",
            "content": "hello!",
            "time": "2018-08-06_13:01:00"
        },
        {
            "sender_name": "Kemmy",
            "sender_email": "qwe@qq.com",
            "content": "how are you!",
            "time": "2018-08-06_14:20:30"
        }
    ],
    "friend_status": "Active",
    "new_friends"=[
        {
            "user_name": "Vaskka",
            "email": "1@1.com"
        },
        {
            "user_name": "Viskka",
            "email": "2@2.com"
        }
    ]
}
</pre>

### 用户注册 (验证email合法性)
#### url
/register/
#### 请求
<pre>
POST
{
    "email": "123456@qq.com"
}
</pre>

#### 回复
<pre>
/*
code : 0 for success
       1 for error
msg  : success
     : error
*/

{
    "code": "0",
    "msg": success,
    "token_code": "2343"
}
</pre>

### 用户登陆
#### url
/sign/
#### 请求
<pre>
POST
{
    "email": "12345@qq.com",
    "password": "admin123"
}
</pre>
#### 回应
<pre>
{
    "code": "0",
    "token": "asdg3GSD9as...",
    "msg": "success",
}
/*
code  0 for success,
      1 for email or password error,
      2 for email already exist and the 'token' is wrong email
*/
</pre>

### 退出登陆
#### url
/logout/
#### 请求
<pre>
POST
{
    "email": "12345@qq.com",
    "token": "qwer"
}
</pre>

#### 回应
<pre>
/*
code 0 for success
     1 for user not sign in
     2 for token is wrong
*/
{
    "code": "0",
    "msg": "success"
}
</pre>

</pre>
